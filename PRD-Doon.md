# [PRD] Doon 제품 요구사항 정의서 (Lean & Core)

## 1. 문서 개요

- **프로젝트/기능명:** Doon - 개인 AI 컴퓨터 오퍼레이터
- **작성일:** 2026년 07월 20일
- **현재 상태:** 초안 (Draft)

---

## 2. 배경 및 비즈니스 목적 (Why)

### 문제 정의 (Problem)

사용자는 하나의 디지털 업무를 끝내기 위해 필요한 정보를 찾고, 여러 화면과 도구를 오가며 내용을 입력하고, 결과를 확인하고, 수정하는 과정을 직접 수행해야 한다. 기존 AI는 답변이나 초안을 생성하는 데에는 강하지만, 사용자의 실제 컴퓨터 환경에서 행동하고 결과물을 완성하는 책임까지 맡지는 못한다.

그 결과 사용자는 AI의 답변을 다시 복사하고, 실행 환경에 옮기고, 세부 작업을 직접 마무리해야 한다. 반복적이고 귀찮은 실행 과정은 여전히 사용자의 시간과 집중력을 소모한다.

### 비즈니스 목적 (Business Goal)

Doon은 사람이 컴퓨터에 직접 명령하고 조작하는 구조를, 사람이 원하는 목적을 전달하면 AI가 컴퓨터 사용을 대신 책임지는 구조로 전환한다.

특정 프로그램이나 정해진 매크로를 자동화하는 도구가 아니라, 사용자의 의도와 현재 환경을 이해하고 복잡한 디지털 업무를 단계적으로 수행하는 개인 컴퓨터 오퍼레이터를 제공한다.

### 기대 효과 (Impact)

- 사용자는 작업 방법이 아니라 원하는 결과를 중심으로 요청할 수 있다.
- 반복적인 입력, 탐색, 전환, 수정에 사용하는 시간을 줄일 수 있다.
- 사용자는 세부 실행이 아니라 의미 있는 단계별 결과만 판단할 수 있다.
- AI의 실행 과정을 확인하고 필요한 시점에 개입해 통제권을 유지할 수 있다.
- 서비스는 개인 구독을 시작으로 사용량 기반 요금제와 조직용 계약으로 확장할 수 있다.

### 핵심 가치 제안 (Value Proposition)

> Doon은 사용자의 의도와 컴퓨터 환경을 이해하고, 복잡하고 귀찮은 디지털 업무를 실제 결과가 완성될 때까지 대신 수행해 사용자가 실행 과정이 아니라 판단과 결과에 집중하도록 돕는다.

### 제품 정의 (Product Definition)

> Doon은 사용자의 컴퓨터와 작업 방식을 이해하고, 한 번의 요청으로 복잡한 디지털 업무를 단계별로 수행해 결과까지 완성하는 개인 AI 컴퓨터 오퍼레이터다.

## 3. 사용자 스토리 (User Story)

- **As a (누가):** 컴퓨터로 여러 단계의 디지털 업무를 수행하는 사용자인 내가
- **I want to (무엇을):** 원하는 결과를 자연어로 요청하고, Doon이 내 컴퓨터 환경에서 필요한 작업을 계획하고 실행하도록 맡기고 싶다.
- **So that (왜):** 반복적이고 귀찮은 실행 과정에서 벗어나 중요한 판단과 최종 결과에 집중할 수 있기 때문이다.

### 보조 사용자 스토리

- 사용자는 Doon이 전체 요청을 의미 있는 단계별 결과물로 나누어 보여주기를 원한다.
- 사용자는 각 단계에서 결과를 승인하거나 수정 방향을 전달하거나 같은 단계를 다시 실행하거나 작업을 중단할 수 있기를 원한다.
- 사용자는 Doon이 허용된 범위 안에서만 컴퓨터와 데이터에 접근하기를 원한다.
- 사용자는 Doon이 무엇을 수행했고 어떤 결과를 만들었는지 확인할 수 있기를 원한다.
- 사용자는 시간이 지날수록 Doon이 자신의 환경과 선호를 반영하기를 원한다.

## 5. 핵심 사용자 경험 흐름

1. 사용자가 데스크톱에 Doon을 설치하고 필요한 접근 권한을 설정한다.
2. 사용자가 현재 작업 중 전역 단축키로 Doon을 호출한다.
3. 사용자가 자연어로 원하는 결과를 요청한다.
4. Doon이 요청과 현재 컴퓨터 맥락을 분석하고 필요한 경우에만 추가 질문을 한다.
5. Doon이 전체 요청을 단계별 결과물 단위로 분해한다. MVP 문서 작업은 `requirements_collected`, `content_drafted`, `document_formatted`, `file_saved` 순서를 사용한다.
6. Doon이 현재 단계의 작업을 실제 컴퓨터 환경에서 수행한다.
7. 사용자가 생성된 단계별 결과물을 승인하거나 수정하거나 다시 실행하거나 중단한다.
8. Doon이 승인된 결과를 바탕으로 다음 단계를 수행한다.
9. 모든 단계가 끝나면 Doon이 완료 조건을 검증하고 최종 결과를 보고한다.

### MVP 문서 작업의 기준 단계

| Stage ID | 사용자에게 제시하는 결과 |
| :--- | :--- |
| `requirements_collected` | Discord 요구사항 수집 및 정리 결과 |
| `content_drafted` | 문서 구조와 내용 초안 |
| `document_formatted` | 한컴독스에 작성하고 형식을 적용한 문서 미리보기 |
| `file_saved` | 승인된 로컬 폴더에 저장하고 검증한 `.hwp` 파일 |

각 단계 결과는 반드시 사용자 검토 상태로 전환한다. 승인 전에는 다음 stage ID를 시작하지 않으며 수정과 재실행은 같은 stage ID의 새 결과를 만든다. `file_saved` 승인 후에만 전체 작업을 완료 처리한다.

## 6. 수익모델 정의

### 기본 수익모델

- **개인 구독:** 작업량, 지원 범위, 개인화 수준에 따른 월간 또는 연간 요금제
- **사용량 기반 과금:** 장시간 작업이나 높은 연산량이 필요한 경우 추가 사용량 제공
- **팀 구독:** 팀 공용 정책, 작업 방식, 권한과 감사 기록 제공
- **기업용 계약:** 중앙 관리, 보안 정책, 사내 환경 연결 및 전용 지원 제공

### 수익모델 원칙

- 광고와 사용자 데이터 판매를 주요 수익원으로 사용하지 않는다.
- 사용자가 처리량과 비용을 예측할 수 있어야 한다.
- 로컬 처리와 개인정보 보호 수준을 유료 가치로 제공할 수 있다.

## 7. 유저 채널 및 사용 환경

### 주요 제품 채널

- **설치 채널:** 공식 웹사이트에서 서명된 데스크톱 앱 다운로드
- **주 실행 환경:** 사용자의 로컬 컴퓨터
- **초기 지원 환경:** macOS 우선 지원
- **호출 채널:** 전역 단축키, 메뉴바 또는 데스크톱 인터페이스
- **상호작용 채널:** 자연어 명령 창, 화면 오버레이, 단계별 결과 확인 화면
- **알림 채널:** 운영체제 알림과 앱 내 작업 상태 알림

### 향후 확장 채널

- 다른 데스크톱 운영체제 지원
- 음성 기반 호출 및 명령
- 모바일 보조 인터페이스를 통한 상태 확인과 승인
- 조직 단위 설치 및 중앙 배포
- 고급 사용자를 위한 패키지 관리자 또는 CLI 설치

### 데스크톱 설치 홈페이지

- 홈페이지는 공식 데스크톱 앱 다운로드의 기본 진입점으로 제공한다.
- 실제 제품 화면과 실행 흐름을 사용해 Doon의 가치를 설명한다.
- 지원 운영체제, 시스템 요구사항, 현재 버전, 코드 서명 및 설치 절차를 명확히 표시한다.
- 권한과 개인정보 처리 방식을 설치 전에 확인할 수 있어야 한다.
- 홈페이지 구현은 데스크톱 제품의 핵심 경험과 배포 파일이 안정된 이후 진행한다.

### 채널 원칙

- 일반 사용자의 기본 경험은 데스크톱 앱 설치와 단축키 호출로 구성한다.
- CLI는 대표 사용자 경험이 아니라 고급 사용자와 개발 환경을 위한 보조 설치 채널로 제공한다.
- 사용자는 현재 작업 환경을 벗어나지 않고 Doon을 호출하고 통제할 수 있어야 한다.

## 8. 제외 스코프 (Out of Scope)

- 모바일 기기만으로 모든 작업을 수행하는 독립형 모바일 제품
- 사용자 승인 없이 위험 행동을 무제한으로 수행하는 완전 자율 실행
- 사용자의 명시적 허용 없는 상시 화면 기록 및 행동 수집
- 광고 목적의 사용자 행동 분석과 개인정보 판매
- 제품 초기 단계에서 모든 운영체제와 모든 컴퓨터 환경을 동시에 지원하는 것
- 제품 초기 단계에서 조직용 중앙 관리와 복잡한 관리자 기능을 완성하는 것
- 제품 초기 단계에서 외부 개발자용 확장 생태계와 마켓플레이스를 구축하는 것

---

## 9. 핵심 성공 기준

- 사용자가 작업 방법이 아니라 원하는 결과를 중심으로 요청할 수 있다.
- 사용자가 직접 수행해야 하는 반복적인 컴퓨터 조작이 실질적으로 감소한다.
- 사용자가 단계별 결과물을 이해하고 승인 또는 수정할 수 있다.
- 사용자가 언제든 실행을 중단하고 통제권을 회수할 수 있다.
- Doon이 작업 완료 여부를 스스로 검증하고 결과를 명확히 보고한다.
- 사용자가 동일하거나 유사한 작업을 다시 Doon에게 맡길 의향을 보인다.

---

## 10. 제품 원칙

1. **결과 중심:** 화면 조작 자체가 아니라 사용자가 요청한 결과의 완성을 목표로 한다.
2. **감독 가능한 자율성:** 사용자의 개입을 최소화하되 단계별 결과에 대한 통제권을 보장한다.
3. **실제 환경에서의 실행:** 답변이나 초안 생성에 머무르지 않고 사용자의 컴퓨터에서 필요한 행동을 수행한다.
4. **최소 권한:** 작업에 필요한 범위만 접근하고 민감한 정보는 가능한 한 로컬에서 처리한다.
5. **설명 가능한 진행:** 사용자가 현재 단계, 결과, 실패 원인과 다음 행동을 이해할 수 있어야 한다.
6. **개인화 가능한 작업 방식:** 사용자의 선택과 환경을 학습하되 학습 범위와 저장 내용을 사용자가 통제한다.
